## Цели
+ Целью задачи является создание робота который будет убирать мусор из зелёных зон города (парки, заповедники). Робот будет ездить строго внутри парка. Робот должен будет осуществлять захват мусора с помощью манипуляторов и складывать мусор в корзину которая присоединяется к нему. Как только корзина заполнится или уровень заряда робота упадёт, робот поедет на станцию зарядки. Пока робот заряжается, человек ассистент выкинет мусор в общий муссорный бак и вернёт чистую корзину на место.
+ Решение будет на языке программирования **C++**.
## Архитектурные решения
+ Материнской платой робота будет считаться **Arduino**. Поскольку обладает лёгкости в программировании и использовании.
+ Робот будет передвигаться через **колёса луноходов**. Это обеспечит роботу дополнительную устойчивость и хорошую проходимость для робота. Он сможет проезжать труднодоступную местность, по типу грязи, луж, льду и т.д.
+ Для захвата мусора у робота будет **манипулятор с клещевой грузозахватной частью**. Это решение обеспечит хороший захват груза и транспортировку его в корзину.
+ Для визуальной орентации будут использоваться **ультрозвуковые датчик**, с помощью них робот сможет видеть и объезжать припятствия, а так же определять на каком расстояни находится объект от робота.
## Контекстная диаграмма
![pictch](https://github.com/blackholazer/Green-Zone-Robot/blob/da2080ff32998de9c0e2061322637e978af3f301/WorkRobotDio.drawio.png)

### На данной диаграмме представлена система логики робота (то как он будет передвигаться и мыслить).
Как только робота включают начинается периуд ожидания, после которого робот начинает процесс **патрулирования**. Если во время **патрулирования** будет обнаружен мусор, включится блок кода отвечающий за **захват** мусора и погрузка его в корзину. Так же во время патрулирования робот будет проверять рессурсы (оставшийся заряд робота и загруженность корзины), если одно из условий будет выполняться робот поедет на базу, для подзарядки и опустошение корзины. Когда робот восполнит заряд, он снова начнёт процесс **поиск мусора**.

## Взаимодейтсвие клиента и сотрудника с роботом
![pictch](https://github.com/blackholazer/Green-Zone-Robot/blob/81035d2e5322c8892a2d61f27abd52135d6b0df4/InteractRob.drawio.png)

На этой диаграме представлена взаимодействие сотрудника тех. поддержки и клиента (гос-во или физическое лицо) с роботом. Разберём её по конкретнее:
### Взаимодейтсвие клиента с роботом:
+ Клиент может совершить **ЗАПУСК** (запустить робота для уборки парка)
+ Клиент может совершить **ЗАВЕРШЕНИЕ** (завершить работу робота чтобы сотрудники тех. поддержки могли совершить работу над ним)
### Взаимодейтсвие сотрудника тех. поддержки с роботом:
+ Тех. поддержка может провести **ПРОВЕРКУ ОБОРУДОВАНИЯ** (для предотвращения поломок в будующем)
+ Тех. поддержка может провести **ПОЧИНКУ СЛОМАННЫХ ЧАСТЕЙ**
+ Тех. поддержка может провести **УСТАНОВКУ ОБНОВЛЕНИЙ** (для улучшения работы робота или исправление ошибок в коде)
